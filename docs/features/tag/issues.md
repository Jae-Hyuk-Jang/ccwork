# 태그 기능 이슈 분해

`prd.md`를 수직 슬라이싱(vertical slicing) 원칙으로 쪼갠 이슈 목록이다. 각 이슈는 레이어(API/상태/UI)별로 나뉘지 않고, 사용자에게 실제로 확인 가능한 가치를 end-to-end로 전달하는 단위로 나눴다. 순서대로 진행하면 각 이슈가 이전 이슈 위에서 독립적으로 데모 가능하다.

---

## 이슈 1: 태그 데이터 모델 추가 및 상세 화면에 읽기 전용 표시

**설명**

`Note` 타입에 `tags?: string[]` 필드를 추가하고, 노트 상세 화면(`NoteEditor`)과 사이드바 목록(`NoteItem`)에서 노트에 태그가 있으면 칩 형태로 표시한다. 이 단계에서는 추가/삭제 UI가 없다 — 이미 태그가 붙어 있는 노트를 열었을 때 확인만 가능하다.

**완료 조건 (Acceptance Criteria)**

- `Note` 인터페이스에 `tags?: string[]` 필드가 존재한다.
- 태그가 있는 노트를 상세 화면에서 열면 태그 칩이 표시된다.
- 태그가 없는 노트를 상세 화면에서 열면 태그 영역에 빈 줄/빈 칩이 생기지 않는다.
- 사이드바 목록(`NoteItem`)에도 동일하게 읽기 전용으로 태그가 표시된다 (`spec-fixed.md` 결정 #1).

**Given / When / Then**

- Given 태그가 `["React", "TypeScript"]`인 노트가 존재하고, When 사용자가 이 노트를 상세 화면에서 열면, Then 태그 칩 "React"와 "TypeScript"가 순서대로 표시된다.
- Given 태그가 없는 노트가 존재하고, When 사용자가 이 노트를 상세 화면에서 열면, Then 태그 영역에 아무 칩도 표시되지 않는다.
- Given 태그가 `["공부"]`인 노트가 존재하고, When 사용자가 사이드바 목록을 보면, Then 해당 노트 카드에 "공부" 칩이 읽기 전용으로 표시된다.

## 이슈 2: 노트 상세 화면에서 태그 추가

**설명**

상세 화면의 태그 입력창에 텍스트를 입력하고 Enter를 누르면 칩으로 즉시 추가된다. 앞뒤 공백은 trim하고, 대소문자를 무시해 이미 있는 태그와 중복되면 추가하지 않는다. 추가된 태그는 로컬 상태로만 존재하다가 '저장' 버튼을 눌러야 서버에 반영된다.

**완료 조건**

- 태그 입력창에 텍스트를 입력하고 Enter를 누르면 칩이 즉시 추가되고 입력창이 비워진다.
- 입력값의 앞뒤 공백은 trim한 뒤 추가된다.
- 이미 존재하는 태그와 대소문자만 다른 값을 입력하면 추가되지 않는다(무시, 에러 없음).
- 공백만 입력한 채 Enter를 누르면 아무 것도 추가되지 않는다.
- '저장'을 누르기 전까지는 서버(json-server)에 반영되지 않는다.
- '저장'을 누르면 추가된 태그가 제목/내용과 함께 한 번에 서버에 반영된다.
- 새 노트를 작성하는 중(아직 저장 전, id 없음)에도 동일하게 태그를 추가할 수 있다.

**Given / When / Then**

- Given 노트 상세 화면이 열려 있고 태그가 비어 있을 때, When 사용자가 입력창에 "React"를 입력하고 Enter를 누르면, Then "React" 칩이 추가되고 입력창은 빈 문자열이 된다.
- Given 노트에 이미 "React" 태그가 있을 때, When 사용자가 입력창에 "react"를 입력하고 Enter를 누르면, Then 태그 목록에는 여전히 "React" 하나만 있다.
- Given 사용자가 태그 "공부"를 추가했지만 아직 '저장'을 누르지 않았을 때, When 서버 데이터를 확인하면, Then "공부"는 아직 반영되어 있지 않다.
- Given 사용자가 태그 "공부"를 추가한 뒤, When '저장' 버튼을 클릭하면, Then 서버에 저장된 노트의 `tags`에 "공부"가 포함된다.
- Given 사용자가 '새 노트' 작성 화면을 열고 아직 저장하지 않은 상태에서, When 태그 "아이디어"를 추가한 뒤 제목을 입력하고 '저장'을 클릭하면, Then 새로 생성된 노트의 `tags`에 "아이디어"가 포함된다.

## 이슈 3: 노트 상세 화면에서 태그 삭제

**설명**

태그 칩에 있는 x 버튼을 클릭하면 확인 절차 없이 즉시 해당 칩이 제거된다. 삭제도 추가와 동일하게 로컬 상태로만 존재하다가 '저장' 버튼을 눌러야 서버에 반영된다.

**완료 조건**

- 각 태그 칩에는 삭제용 x 버튼이 있다.
- x 버튼을 클릭하면 확인 다이얼로그 없이 즉시 해당 태그가 화면에서 제거된다.
- '저장'을 누르기 전까지는 서버에 반영되지 않는다.
- '저장'을 누르면 삭제된 상태가 서버에 반영된다.

**Given / When / Then**

- Given 노트에 "React", "TypeScript" 태그가 있을 때, When 사용자가 "React" 칩의 x 버튼을 클릭하면, Then "React" 칩은 즉시 사라지고 "TypeScript" 칩만 남는다.
- Given 사용자가 태그를 삭제했지만 아직 '저장'을 누르지 않았을 때, When 서버 데이터를 확인하면, Then 삭제 전 태그 목록이 그대로 남아 있다.
- Given 사용자가 "React" 태그를 삭제한 뒤, When '저장' 버튼을 클릭하면, Then 서버에 저장된 노트의 `tags`에는 "React"가 더 이상 없다.

## 이슈 4: 노트 편집 취소 시 태그 변경 롤백

**설명**

태그를 추가하거나 삭제한 뒤 '저장'이 아니라 '취소'를 누르면, 제목/내용과 마찬가지로 태그 변경도 전부 원래 상태로 되돌아간다. 같은 노트를 계속 편집 중인 상태로 '취소'를 눌러도(에디터가 리마운트되지 않아도) 동일하게 동작해야 한다.

**완료 조건**

- '취소'를 누르면 추가했던 태그가 화면에서 사라지고 원래 태그만 남는다.
- '취소'를 누르면 삭제했던 태그가 다시 화면에 나타난다.
- '취소' 이후 서버에는 애초에 아무 태그 변경도 반영되지 않는다.
- 같은 노트가 계속 선택된 상태로 '취소'를 눌러도 위 동작이 동일하게 적용된다.

**Given / When / Then**

- Given 노트에 "React" 태그만 있고, When 사용자가 "TypeScript"를 추가한 뒤 '취소'를 클릭하면, Then 태그 목록에는 "React"만 남는다.
- Given 노트에 "React", "TypeScript" 태그가 있고, When 사용자가 "TypeScript"를 삭제한 뒤 '취소'를 클릭하면, Then 태그 목록에 "React"와 "TypeScript"가 다시 표시된다.
- Given 사용자가 같은 노트를 계속 편집 중이고 태그를 추가한 뒤, When '취소'를 클릭하고 같은 노트를 다시 확인하면, Then 서버에도 화면에도 추가했던 태그가 반영되어 있지 않다.
