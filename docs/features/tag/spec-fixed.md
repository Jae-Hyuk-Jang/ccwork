# 태그 기능 정의서 (확정)

`spec-original.md`의 기본 요구사항에 대해, 데이터 구조·UI 패턴·엣지 케이스를 질문/답변으로 확정한 버전이다.

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항 (원본)

- 노트에 태그를 추가할 수 있다
- 노트에 추가된 태그를 삭제할 수 있다
- 태그 목록을 노트 상세 화면에서 확인할 수 있다

## 확정된 설계 결정

| #   | 항목           | 결정                                                                                                                                                                                                                | 근거                                                                                                                                                                                                 |
| --- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 기능 범위      | 태그 **추가/삭제는 `NoteEditor`(상세 화면)에서만** 가능하다. 사이드바 목록(`NoteItem`)은 기존처럼 **읽기 전용 표시만** 유지한다.                                                                                    | 스펙이 "상세 화면에서 확인"만 명시. 목록에 편집 UI까지 넣으면 좁은 카드 안에서 UX가 지저분해지고 로직이 두 곳에 중복된다.                                                                            |
| 2   | 입력 UX        | 텍스트 입력 필드에 태그명을 입력하고 **Enter를 누르면 즉시 추가**되며 입력창은 비워진다.                                                                                                                            | Notion/Gmail 류의 익숙한 태그 입력 패턴. 별도 "추가" 버튼 클릭 없이 키보드만으로 연속 입력 가능.                                                                                                     |
| 3   | 중복 판정 기준 | 태그 문자열은 **앞뒤 공백을 trim하고 대소문자를 무시**하여 비교한다 (`'React'`와 `'react'`는 중복으로 간주). 저장/표시는 사용자가 입력한 원문 그대로 유지한다.                                                      | 스펙의 "중복 태그는 허용하지 않는다"만으로는 대소문자 처리 기준이 없어, 실수로 비슷한 태그를 중복 생성하는 것을 막기 위해 관대한 쪽(대소문자 무시)으로 정함.                                         |
| 4   | 개수/길이 제한 | 태그 개수와 글자수에 **제한을 두지 않는다**.                                                                                                                                                                        | 스펙에 없는 제약을 임의로 추가하지 않는다. UI만 자연스럽게 줄바꿈/스크롤되도록 처리.                                                                                                                 |
| 5   | 저장 시점      | 태그도 제목/내용과 동일하게 `NoteEditor`의 **로컬 state로만 관리**하다가, **'저장' 버튼 클릭 시** `createNote`/`updateNote` 호출과 함께 서버에 반영된다. 즉시 API 호출(태그 추가/삭제마다 서버 반영)은 하지 않는다. | 이 프로젝트의 기존 패턴(제목/내용도 로컬 state → 저장 버튼에서 한 번에 반영)과 동일하게 맞춰야 '취소' 버튼이 태그 변경도 함께 되돌릴 수 있고, 새 노트 작성 중(아직 id 없음)에도 태그를 붙일 수 있다. |
| 6   | 삭제 UI        | 각 태그 칩(뱃지) 안에 **x 버튼**을 표시하고, 클릭하면 바로 제거된다 (확인 다이얼로그 없음).                                                                                                                         | `NoteItem`의 개별 삭제 버튼과 동일한 즉시-삭제 패턴. 태그는 되돌리기 쉬운(다시 입력하면 그만인) 가벼운 데이터라 확인 단계가 과함.                                                                    |
| 7   | 배치 위치      | `NoteEditor` 레이아웃에서 태그 입력/칩 영역은 **제목 입력과 내용 textarea 사이**에 위치한다.                                                                                                                        | 메타데이터(태그)가 본문(내용)보다 먼저 오는 배치가 자연스럽고, 사이드바 `NoteItem`의 "제목 → 내용 → 태그" 시각적 흐름과도 크게 어긋나지 않는다.                                                      |

## 용어 정의 (Ubiquitous Language)

인터뷰 과정에서 고정한 용어다. 이후 코드·문서·대화에서 아래 단어들은 항상 같은 것을 가리킨다 (`prd.md`의 "용어 정의"도 이 정의를 그대로 따른다).

| 용어            | 정의                                                                    |
| --------------- | ----------------------------------------------------------------------- |
| 태그 (Tag)      | 노트에 붙이는 자유 형식 문자열 라벨. `Note.tags?: string[]`로 저장한다. |
| 태그 칩 / 뱃지  | 태그를 화면에 표시하는 pill 모양 UI 요소 (`NoteTagBadge` 컴포넌트).     |
| 중복 태그       | 앞뒤 공백을 trim하고 대소문자를 무시했을 때 문자열이 동일한 태그.       |
| 상세 화면       | 노트를 보거나 편집하는 화면 (`NoteEditor` 컴포넌트).                    |
| 목록 / 사이드바 | 노트 목록을 보여주는 화면 (`NoteList`/`NoteItem` 컴포넌트).             |

## 비고 (원본 유지)

- 태그는 문자열로 저장한다 (`Note.tags?: string[]`)
- 중복 태그는 허용하지 않는다 (판정 기준은 위 3번 참고)
