2. 데이터 흐름

요청/응답이 실제로 오가는 경로. json-server ↔ api 계층 ↔ Context ↔ UI 컴포넌트 순으로 CRUD 호출과 응답이 이동하는 방향을 화살표로 표현했다. 

flowchart LR
  Server[("json-server<br/>:3001/notes")]

  subgraph ApiLayer["api/notes.ts"]
    Fetch["fetchNotes()"]
    Create["createNote()"]
    Update["updateNote()"]
    Delete["deleteNote()"]
  end

  subgraph CtxLayer["NotesContext"]
    CtxState["notes / loading / error"]
  end

  subgraph UILayer["UI 컴포넌트"]
    List["NoteList"]
    Item["NoteItem"]
    Editor["NoteEditor"]
  end

  Server -- "마운트 시 1회" --> Fetch --> CtxState
  Editor -- "저장" --> Create --> Server
  Editor -- "저장" --> Update --> Server
  Item -- "삭제" --> Delete --> Server
  Create -- "응답으로 배열에 추가" --> CtxState
  Update -- "응답으로 항목 교체" --> CtxState
  Delete -- "성공 시 필터링" --> CtxState
  CtxState --> List
  CtxState --> Editor
  List --> Item

3. 상태 맵

상태가 "어디에" 있고 "누가 소유"하는지에 집중한 지도. 전역 상태(Context, 파란색 계열)와 로컬 UI 상태(컴포넌트 useState, 노란색 계열)를 색으로 명확히 구분했다.
